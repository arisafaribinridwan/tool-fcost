Architecture

Arsitektur aplikasi: UI tipis, service layer tebal

Project ini memakai pemisahan tanggung jawab yang cukup jelas. UI bertugas menerima input user dan menampilkan status. Service layer menjalankan business logic. Utility layer menangani hal pendukung seperti path safety dan integrasi OS.

Peta layer

Entry point: run.py memanggil ensure_runtime_dirs() lalu run_desktop_app(paths).
Runtime paths: app/__init__.py mendefinisikan AppPaths, get_runtime_root(), dan folder configs, masters, uploads, outputs.
UI layer: app/ui/main_window.py dan app/ui/settings.py.
Service layer: app/services/* memegang validasi, import, registry job, preflight, pipeline, recipe, transform, output.
Utility layer: app/utils/* untuk dialog OS, path safety, log sanitizer, runtime guardrails.

Modul service penting

ModulTanggung jawab
source_service.pyValidasi source .xlsx/.csv, load DataFrame, cek kolom wajib, copy source ke uploads/.
dataframe_io_service.pyMembaca CSV/Excel dengan pesan error yang ramah. Nama sheet Excel dicocokkan case-insensitive.
config_service.pyDiscover, load, normalisasi, dan validasi YAML. Mendukung schema klasik dan step recipe.
job_profile_service.pyMembaca/menulis configs/job_profiles.yaml, validasi registry, ekstraksi dependensi master.
preflight_service.pyCek source dan config sebelum tombol execute diaktifkan.
pipeline_service.pyOrkestrator utama: load config, copy source, baca source, jalankan transform/recipe, tulis output.
recipe_service.pyEngine untuk recipe step-based seperti extract sheet, derive column, lookup rules, summary FCOST.
transform_service.pyEngine config klasik: master lookup, transform conditional/formula/filter, pivot/group_by output.
output_service.pyMenulis workbook .xlsx, sanitasi nama sheet, styling, header, periode, freeze pane.
import_service.pyCopy config/master dari luar ke folder runtime dengan suffix jika nama bentrok.

Design choice yang perlu dipertahankan

Project ini sengaja membatasi path runtime. Config harus berada di configs/, master di masters/, source yang diproses dicopy ke uploads/, dan output di outputs/. Jangan membuat fitur yang langsung menerima path bebas untuk master/output tanpa melewati helper path safety.

UI juga tidak menjalankan pipeline di thread utama. Di main_window.py, eksekusi dilakukan lewat Thread dan komunikasi balik ke UI memakai Queue. Ini penting agar window tidak freeze saat pandas/openpyxl bekerja.

Belajar konsep: “separation of concerns” berarti setiap file punya tugas spesifik. Jika semua logic ditaruh di UI, kode cepat sulit dites. Dengan service layer, test bisa memanggil function langsung tanpa membuka window desktop.

Data type domain

pipeline_types.py berisi dataclass kecil seperti PipelineResult, PipelineStepStatus, PreflightFinding, dan PreflightResult. Dataclass membuat data hasil proses lebih eksplisit daripada dictionary acak.

PipelineResult:
  output_path: Path
  source_copy_path: Path
  sheets_written: int
  duration_ms: int | None
