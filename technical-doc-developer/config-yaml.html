YAML config

Memahami registry job, config klasik, dan step recipe

Kekuatan tool ini ada pada pendekatan config-driven. Banyak perubahan proses bisa dilakukan di YAML tanpa mengubah kode Python, selama masih mengikuti schema yang didukung validator.

Registry job: configs/job_profiles.yaml

Registry menghubungkan nama pekerjaan di UI dengan file config yang dipakai engine. Hanya job yang enabled, valid, dan menunjuk ke config valid yang muncul di selector utama.

jobs:
- id: fcost-lcd-import
  label: FCOST LCD IMPORT
  config_file: monthly-report-recipe-lcd-import.yaml
  enabled: true
- id: fcost-lcd-seid
  label: FCOST LCD SEID
  config_file: monthly-report-recipe.yaml
  enabled: true

job_profile_service.py memvalidasi bahwa id, label, config_file adalah string non-kosong, enabled boolean, dan tidak ada duplikasi id/label.

Mode config klasik

Config klasik minimal memiliki name, source_sheet, header, dan outputs. Mode ini cocok untuk input yang tabular dan transformasinya tidak terlalu bercabang.

name: "Laporan Tes"
source_sheet: "Sheet1"
header:
  title: "Laporan Tes"
  period_from_column: "tanggal"
masters:
  - file: "masters/produk.csv"
    key: "kode_produk"
    columns:
      - "nama_produk"
      - "kategori"
transforms:
  - type: "formula"
    target: "total"
    operation: "multiply"
    left: "qty"
    right: "harga"
outputs:
  - sheet_name: "Detail"
    columns: ["tanggal", "kode_produk", "nama_produk", "kategori", "qty", "total"]
  - sheet_name: "Summary"
    group_by:
      by: "kategori"
      aggregations:
        total: "sum"

Master strategy klasik

Strategy yang didukung adalah lookup, ordered_rules, dan lookup_rules. Lookup biasa mirip VLOOKUP/join. Ordered rules dan lookup rules cocok untuk mapping berbasis kombinasi kondisi atau teks.

Transform klasik

TypeFungsi
ensure_optional_columnsMenambah kolom jika belum ada agar output tetap stabil.
filter_rowsMenyaring baris dengan operator seperti equals, contains, in, gt/gte/lt/lte, blank/not blank.
formulaMenghitung kolom baru dengan add, subtract, multiply, divide; termasuk validasi numerik dan proteksi divide by zero.
conditionalMenulis nilai berdasarkan kondisi if/elif/else.

Mode step recipe

Recipe step-based dikenali jika payload punya datasets dan steps. Ini dipakai oleh config monthly report yang lebih kompleks.

name: "Monthly Report Final Recipe"
datasets:
  working_dataset: "result"
  canonical_columns:
    - "notification"
    - "model_name"
    - "labor_cost"
steps:
  - id: "sub_1_copy_gqs"
    type: "extract_sheet"
    sheet_selector:
      contains: "GQS"
      case_sensitive: false
    header_locator:
      type: "required_columns"
      scan_rows: [1, 15]
      required: ["Notification", "Category", "Labor Cost"]
    select:
      "Notification": "notification"
      "Model Name": "model_name"
      "Labor Cost": "labor_cost"
    write_to: "result"
outputs:
  - sheet_name: "result"
    columns: ["notification", "model_name", "labor_cost"]

Recipe bisa memakai expression seperti substring, add, divide, ceil, date_diff_days, case, dan runtime_value. Untuk monthly report, UI dapat meminta input periode YYYYMM jika config memiliki ui.period_prompt.enabled: true.

Output workbook

Setiap item di outputs minimal punya sheet_name dan salah satu rule output: columns, pivot, group_by, atau summary. Nama sheet disanitasi agar aman untuk Excel: karakter invalid diganti, panjang maksimal 31 karakter, dan duplikasi diberi suffix.

summary dipakai untuk builder khusus di recipe service, misalnya static_part_pivot_summary, part_pivot_summary, panel_model_summary, dan summary FCOST lain.

Aturan path penting. Master file harus ditulis sebagai path relatif di bawah masters/, misalnya masters/master_table.xlsx. Jangan memakai ../, absolute path, atau drive path Windows.
