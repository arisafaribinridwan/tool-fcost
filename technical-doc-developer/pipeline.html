Pipeline

Dari tombol Execute sampai file Excel keluar

Pipeline adalah rangkaian proses yang dijalankan setelah user memilih source dan job. Orkestrator utamanya adalah run_pipeline() di app/services/pipeline_service.py.

Alur utama run_pipeline()

  1. Validasi source. validate_source_file() memastikan file ada, bukan folder, ekstensi didukung, dan tidak kosong.
  2. Load config. load_config_payload() membaca YAML dan menjalankan validator.
  3. Copy source. Source di luar folder upload dicopy ke uploads/ dengan timestamp sebagai audit trail.
  4. Pilih engine. Jika payload punya root steps dan datasets, pipeline memakai step recipe. Jika tidak, pipeline memakai mode config klasik.
  5. Build output sheets. Engine menghasilkan dictionary {sheet_name: DataFrame}.
  6. Write workbook. write_output_workbook() membuat file .xlsx di outputs/.
  7. Return result. Pipeline mengembalikan PipelineResult berisi path output, path source copy, jumlah sheet, dan durasi.

Mode 1: config klasik

Mode klasik dipakai untuk flow yang relatif umum: baca satu source sheet, enrich dengan master, apply transform, lalu buat output detail/pivot/group by.

source file
  → load_source_dataframe()
  → validate_required_source_columns()
  → apply_master_lookups()
  → apply_transform_steps()
  → build_output_sheets()
  → write_output_workbook()

Transform klasik mendukung ensure_optional_columns, filter_rows, formula, dan conditional. Output klasik mendukung select columns, pivot, dan group by aggregation.

Mode 2: step recipe

Mode step recipe dipakai untuk workflow yang lebih kompleks, misalnya monthly report FCOST yang mengambil sheet berbeda, mendeteksi header dinamis, append dataset, menghitung kolom turunan, lookup rules, resolve duplicate notification, dan membuat summary khusus.

source workbook
  → execute_step_recipe()
  → extract_sheet / derive_column / update_columns / lookup_*
  → duplicate_group_rewrite / map_ranges
  → summary output builders
  → write_output_workbook()

Step yang didukung validator: extract_sheet, derive_column, duplicate_group_rewrite, lookup_exact, lookup_exact_replace, lookup_rules, map_ranges, dan update_columns.

Preflight di UI

UI tidak langsung mengaktifkan tombol execute. Setelah source dan job dipilih, run_preflight() memvalidasi source dan config. Hasilnya berupa PreflightResult. Tombol execute hanya aktif jika can_execute bernilai true.

Di halaman Settings, run_settings_precheck() mengecek config dan referensi master dari root masters maupun dari steps[].master.file. Ini membantu user menemukan config invalid sebelum dipakai di workflow utama.

Error handling

Service layer memakai ValueError untuk kegagalan validasi detail, lalu pipeline membungkusnya sebagai PipelineError untuk pesan domain yang bisa ditampilkan ke user. Di UI, pesan error disanitasi agar tidak membocorkan path internal yang membingungkan.

Belajar konsep: pandas DataFrame bisa dibayangkan seperti tabel Excel di memory. Setiap transform mengubah tabel ini sedikit demi sedikit sampai siap ditulis kembali menjadi workbook.
