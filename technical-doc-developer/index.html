Panduan utama

Technical documentation untuk developer dan pemula

Project ini adalah aplikasi desktop automasi Excel/CSV. User memilih source, memilih job, lalu engine membaca YAML config untuk mengubah data dan menulis output workbook .xlsx. Mini situs ini dibuat agar developer bisa memahami kode, dan pemula bisa belajar konsep programming dari contoh nyata.

PythonCustomTkinterpandasopenpyxlYAML

Gambaran besar tool

Tool ini memindahkan pekerjaan Excel manual yang berulang ke proses yang lebih konsisten. Alih-alih user melakukan copy-paste, filter, lookup, rumus, dan summary secara manual, aturan proses ditulis dalam file YAML di folder configs/. Data master diletakkan di masters/. Source yang dipilih user diproses dan hasilnya disimpan ke outputs/.

1. Input — user memilih file .xlsx atau .csv.
2. Job — UI membaca registry configs/job_profiles.yaml untuk menampilkan pekerjaan valid.
3. Config — engine membaca YAML config yang berisi aturan transformasi.
4. Processing — service layer memakai pandas untuk data frame, lookup master, transform, dan summary.
5. Output — openpyxl menulis workbook multi-sheet dengan styling.

Untuk developer

Mulai dari Arsitektur, lalu lanjut ke Alur Pipeline. Dua halaman itu menjelaskan file penting, dependency antar layer, dan alur end-to-end dari UI sampai output.

Untuk pemula

Mulai dari Jalur Belajar Pemula. Di sana konsep seperti function, class, dataclass, exception, thread, YAML, DataFrame, dan testing dijelaskan memakai contoh dari project ini.

Untuk config author

Baca Config YAML. Halaman itu menjelaskan registry job, mode config klasik, mode step recipe, master lookup, transform, dan output sheet.

Daftar folder penting

Folder/filePeran
run.pyEntry point aplikasi. Memastikan folder runtime tersedia lalu menjalankan desktop UI.
app/ui/Layer tampilan desktop. main_window.py untuk workflow utama, settings.py untuk manajemen job/config/master.
app/services/Business logic. Di sinilah source dibaca, config divalidasi, pipeline dijalankan, transform diterapkan, dan output ditulis.
app/utils/Helper lintas OS, keamanan path, sanitasi log, dan dialog file.
configs/Registry job dan file YAML recipe/config runtime.
masters/Workbook/CSV master untuk lookup. Project saat ini tetap memakai Excel master runtime.
tests/Test pytest untuk service, UI behavior, path safety, recipe, transform, output, dan pipeline.
