Getting started

Setup environment, menjalankan app, lint, test, dan build

Halaman ini adalah checklist praktis untuk menyiapkan project di komputer developer. Project menargetkan Python 3.14.x dan memakai virtual environment agar dependency tidak bercampur dengan Python global.

Setup Windows PowerShell

py -3.14 -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
python run.py

Jika memakai CMD, aktifkan environment dengan .venv\Scripts\activate.bat. Setelah aktif, prompt terminal biasanya menampilkan (.venv).

Setup Linux/macOS

python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install --upgrade pip
pip install -r requirements.txt
python run.py
Catatan Linux. App desktop membutuhkan Tk/Tcl. Pada Ubuntu/Mint biasanya perlu paket python3-tk. File picker native bisa memakai zenity atau kdialog, tetapi aplikasi akan fallback ke dialog Tk jika helper itu tidak ada.

Command harian developer

KebutuhanCommandKapan dipakai
Jalankan UI utamapython run.pySaat ingin mencoba flow user dari pilih source sampai output.
Jalankan settingspython -m app.ui.settingsSaat menguji manajemen job/config/master secara langsung.
Lintpython -m ruff check .Sebelum commit atau setelah refactor.
Semua testpython -m pytest -qSebelum menganggap perubahan aman.
Test satu filepython -m pytest tests/test_pipeline_service.py -qSaat sedang fokus di area pipeline.
Test satu casepython -m pytest tests/test_pipeline_service.py::test_run_pipeline_happy_path_csv_with_master_and_pivot -qSaat debugging test tertentu.

Dependency utama

customtkinter dipakai untuk UI desktop modern. pandas adalah engine data frame untuk membaca, filter, merge, group by, dan membuat output data. openpyxl dipakai untuk membaca/menulis Excel dan styling workbook. PyYAML membaca config YAML. pyinstaller membangun bundle portable. pytest dan ruff menjaga kualitas kode.

Build portable

Build harus dilakukan di OS target. Jangan mengandalkan PyInstaller cross-build untuk membuat binary OS lain.

Windows

.\packaging\windows\build.ps1
# atau
.\packaging\windows\build.ps1 -PythonExe "C:\Path\To\python.exe"

Linux

sudo apt-get update
sudo apt-get install -y python3-tk python3.12-venv zenity kdialog
./packaging/linux/bootstrap-build-env.sh
./packaging/linux/build.sh
./packaging/linux/smoke-test.sh

Hasil build Linux berada di dist/ExcelAutoTool/ dan archive dist/ExcelAutoTool-linux-x86_64.tar.gz. Hasil build Windows berada di dist\ExcelAutoTool\.

Belajar konsep: virtual environment adalah “kotak dependency” khusus project. Dengan venv, versi pandas/openpyxl milik project ini tidak mengganggu project Python lain.
