Master Symptom Guide

Panduan Lengkap Regex untuk Sheet symptom

Tujuan panduan ini adalah membantu kamu merapikan master symptom di Excel tanpa memecah file master yang sudah ada. File master_table.xlsx tetap bisa dipakai seperti sekarang, dan yang berubah hanya desain serta isi sheet symptom.

Python regex Search match Priority-based Excel-friendly

Tujuan

Sheet symptom dipakai untuk menerjemahkan kombinasi:

  • part_name
  • isi komentar kerusakan, misalnya symptom_comment

menjadi nilai output symptom.

Rule akan dievaluasi berdasarkan priority. Semakin kecil angkanya, semakin dulu diperiksa. Dengan begitu, rule yang lebih spesifik bisa didahulukan dari rule yang lebih umum.

Struktur Sheet

Di file master_table.xlsx, cukup ubah sheet symptom agar memakai header berikut:

priority | part_name | match_type | pattern | symptom | notes
Catatan: file Excel tetap boleh berisi banyak sheet master lain seperti part_list, factory, branch, action, dan defect_category. Yang perlu berubah hanya sheet symptom.

Arti Kolom

Kolom Fungsi Petunjuk Isi
priority Menentukan urutan pemeriksaan rule. Pakai angka kecil untuk rule yang lebih spesifik. Rekomendasi: kelipatan 10 seperti 10, 20, 30.
part_name Membatasi rule berdasarkan part yang sudah teridentifikasi. Contoh: PANEL, MAIN_UNIT, POWER_UNIT. Penulisannya harus konsisten.
match_type Menentukan cara membaca pattern. Untuk desain final, paling rapi jika distandarkan ke regex.
pattern Pola yang dicek terhadap isi komentar kerusakan. Isi dengan regex Python yang sederhana dan mudah dirawat.
symptom Output hasil klasifikasi. Contoh: LINE, BLANK, TOTAL_OFF.
notes Catatan untuk manusia. Tidak dipakai mesin. Gunakan untuk menjelaskan alasan rule dibuat.

Cara Isi yang Benar

Urutan Kerja

  1. Kelompokkan rule per part_name.
  2. Tulis rule yang paling spesifik dulu.
  3. Taruh rule yang lebih umum di bawahnya.
  4. Taruh rule default atau fallback paling bawah.
  5. Pastikan priority unik dan urut.

Kenapa Ini Penting

Kalau rule umum diletakkan terlalu atas, rule itu bisa "memakan" kasus yang seharusnya masuk ke rule yang lebih spesifik.

Contoh buruk: MATI diletakkan di atas LAYAR MATI atau MATI TOTAL.

Prinsip utama: semakin spesifik pattern-nya, semakin kecil angka priority-nya.

Standar Regex yang Disarankan

Kalau kamu akan mencari referensi dengan keyword seperti regex python, itu sudah tepat. Supaya master tetap mudah dirawat tim operasional, standar internalnya sebaiknya dibatasi ke bentuk yang sederhana saja.

Aturan Rekomendasi
Engine Python re
Semantik match search, bukan fullmatch
Huruf Lebih aman pakai huruf kapital semua jika source dinormalisasi ke kapital
Alternatif kata Pakai |, misalnya GARIS|LINE
Spasi fleksibel Pakai \s+, misalnya DEAD\s+PIXEL
Kata utuh Pakai \b jika perlu, misalnya \bHDMI\b
Fallback default Pakai .* dan letakkan paling bawah
Hindari regex yang terlalu rumit. Jangan mulai dengan lookbehind kompleks, nested group yang sulit dibaca, atau pola yang terlalu "pintar". Fokuskan pattern untuk kebutuhan praktis matching symptom saja.

Contoh Pattern Praktis

Kebutuhan Contoh Pattern Keterangan
Alternatif kata GARIS|LINE Cocok untuk beberapa kata yang maknanya setara.
Spasi bervariasi DEAD\s+PIXEL Tetap cocok walau ada lebih dari satu spasi.
Kata utuh \bHDMI\b Mengurangi risiko cocok ke kata yang lebih panjang.
Variasi singkatan/typo REMOTE|REMOT|SENSOR Bisa menangkap variasi kata umum di lapangan.
Frasa panjang TIDAK\s+ADA\s+GAMBAR|TDK\s+ADA\s+GAMBAR|NO\s+PICTURE Gabungkan beberapa frasa ekuivalen dalam satu baris.
Default .* Dipakai hanya untuk fallback terakhir.

Do dan Don't

Do

  • Pakai penulisan part_name yang konsisten.
  • Berikan jarak angka priority agar mudah sisip rule baru.
  • Mulai dari regex yang sederhana.
  • Masukkan typo umum lapangan jika memang sering muncul.
  • Isi notes singkat agar mudah direview tim lain.

Don't

  • Jangan letakkan rule umum di atas rule spesifik.
  • Jangan campur struktur lama dan baru dalam sheet yang sama.
  • Jangan kosongkan kolom pattern atau symptom.
  • Jangan pakai regex yang terlalu rumit jika belum benar-benar perlu.
  • Jangan ubah nama part seenaknya jika mapping upstream belum ikut berubah.

Checklist Review Sebelum Dipakai

  • Header sheet sudah persis: priority, part_name, match_type, pattern, symptom, notes.
  • Semua priority berupa angka dan urut logis.
  • Semua part_name valid dan konsisten.
  • Semua match_type sesuai standar yang dipilih, idealnya regex.
  • Semua pattern terisi.
  • Semua symptom terisi.
  • Rule spesifik berada di atas rule umum.
  • Fallback seperti .* hanya ada di baris paling bawah untuk part terkait.
  • notes cukup jelas untuk menjelaskan alasan bisnis rule dibuat.

Panduan ini disusun agar file master Excel tetap bisa dipakai sebagai satu sumber master, tanpa perlu memecah sheet master ke file terpisah. Jika kamu ingin, langkah berikutnya yang paling natural adalah membuat versi print/PDF atau menambahkan contoh rule yang khusus mengikuti data symptom real milikmu.
